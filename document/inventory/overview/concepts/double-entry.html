    复式库存没有入库，出库(产品的disparition)或者变换。相反的，所有的移动都是在不同的库位之间移动(可以是虚拟的)。

    操作

    库存调拨表示了货物或者材料在不同的库位之间的移动。

    生产订单
    消耗
    2个车轮：库存 → 产线
    1个车架：库存→产线
    生产：

    1个循环：产品 → 库存

    配置：
    库存：生产订单的物流的原始位置
    产线：在产品表单上的**生产库位**字段
    直运

    1辆自行车:供应商→客户

    配置：
    供应商：在产品表单上
    客户：在销售订单上
    客户发货
    调拨

    1个循环：库存 → 拣货区域

    包装

    1辆自行车:包装区域→出货区域

    发货

    1辆自行车:出货区域→客户

    配置：
    仓库的拣货＋包装＋发货路径
    仓库内部的调拨
    转移：
    1辆自行车:仓库1→中转区
    1辆自行车:中转区→仓库2
    配置：
    仓库2：调拨的源发库位
    仓库1：在中转路径上
    坏掉的产品(报废)

    1辆自行车:仓库→报废

    配置：

    报废：在报废的时候需要的报废库位

    库存
    存货产品丢失

    1辆自行车:仓库→盘点库位

    存货中的额外产品

    1辆自行车:盘点库位→仓库

    配置：

    盘点库位：产品上的"盘点库位"字段

    接收
    1辆自行车:供应商库位→收货库位
    1辆自行车:收货库位→库存库位
    配置：
    供应商库位：采购订单的补货库位
    收货区域：采购订单上的“目标”库位

    分析

    存货分析可以使用产品数量或者产品价值(=产品数量*产品成本)。

    对于每一个盘点库位，多个数据点可以被分析：

    • inventory valuation
    • value creation (difference between the value of manufactured products and the cost of raw materials used during manufacturing) (negative)
    • value of lost/stolen products
    • value of scrapped products
    • value of products delivered to clients over a period
    • value of products received from suppliers over a period (negative)
    • value of products in transit between locations

    补货＆拉规则

    补货就是一个特定库位的特定产品的特定数量的需求。可以手工创建或者自动的触发于：

    新的销售订单
    工作

    补货是在客户库位上创建的客户订单上的产品(需要发货给客户的)

    配置

    补货库位：在客户信息上的字段"客户库位"(属性)

    最小库存规则
    工作

    补货在规则位置创建。

    配置

    补货库位：在该规则上，字段"库位"

    补货规则
    工作

    新的补货是在规则上的源库位上创建的

    [UNKNOWN NODE problematic]拉规则*描述了特定库位上的补货如何实现，例如：

    • 产品移动的源头位置(源库位 )

    • 要么补货是：[UNKNOWN NODE title_reference] or MTS

    路径

    路线中对拉规则进行分组。路线定义了产品的路径。根据产品/销售订单行／仓库，……路线可能会用也可能不能用。

    要完成一个补货，系统会搜索路线中定义的规则(根据顺序或者优先级)：

    仓库

    仓库路线示例：拣货→ 包装→ 发货

    调拨列表

    拣货区域→ 包装区域

    包装列表：

    包装区域→ 门A

    交货单：

    门A→ 客户

    描述如何组织你的仓库的路线应该在仓库中定义。

    一个产品

    产品路径例子：质量控制

    收货：

    供应商 → 收货

    确认：

    收货区域 → 质检区域

    库存：

    质检 → 库存

    产品种类

    产品类别路线示例：cross-dock

    收货：

    供应商 → 收货

    Cross-Docks:

    收获区域 →出货区域

    发货：

    发货区域 → 客户

    销售订单行

    销售订单行示例：直运

    订单：

    供应商 → 客户

    推规则

    当产品进入一个特定库位的时候推规则会触发。它们会把产品自动移动一个新的库位。是否要用推规则依赖于可用的路线。

    质量控制

    • 产品进入收货区

    • 推规则1：收货区→ 质量控制

    • 推规则2： 质量控制→库存

    仓库中转区

    • 产品进入中转区

    • 推：中转区 → 仓库 2

    补货组

    路线和规则定义库存的移动，对于每一个规则，都需要提供一个操作类型：

    • 拣货

    • 包装

    • 发货单

    • 采购订单

    • ...

    如果拉规则和库位一样的话，库存调拨就可以汇总到同一类型的操作类型中

    销售订单创建一个补货组，所以同一个销售订单的拣货和发货单被分开。但是你可以在再定点规则上定义特定的组(例如，可以把特定产品的采购询价单汇总起来)。
