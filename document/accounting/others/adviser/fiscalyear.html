    在关闭一个财政年度前, 需要做几步以确保你的会计是正确的,最新的,和精确的:

    • 确保你完全核销了* 银行账户(s) [UNKNOWN NODE problematic]直至年底, 同时确认登帐期末余额和银行对账单的期末余额一致。

    • 确认所有**客户发票**已输入和批准。

    • 确认您已输入并同意的所有 供应商账单。

    • 审核所有的费用，保证其正确。

    • 证实,所有的 [UNKNOWN NODE problematic]received payments**已登记并准确记录。

    年终（科目）余额

    • 运行 税报表, 并验证你的税务信息是正确的。

    • 在 资产负债表 调节所有科目 :

      • 将银行对账单和实际的银行余额一致, 可利用* Bank Reconciliation [UNKNOWN NODE problematic]报表来辅助。

      • 通过运行**Aged Receivables** and [UNKNOWN NODE problematic]Aged Payables**报表, 核销所有交易中的现金和银行科目。

      • 检查你的科目, 确定理解所有交易对科目的影响和交易的性质,确保包括贷款和固定资产。

    如果你的会计正经历今年年底审计,他们可能想要复印纸质的所有资产负债表项目(如贷款、银行账户、预付款、销项税,等等…), 这些应与YuanCloud内的余额一致。

    在这个过程中, 很好的实用方法就设置会计年度的最后一天, 通过设置 Lock date for Non-Advisers , 这个在会计配置下设置。这样, 在审计时, 会计可确保没有人能改变前一年的交易。

    关闭会计年度

    在YuanCloud, 不需要做一个特定的年底关闭凭证以关闭损益表科目。这些报表实时产生,这意味着* 损益表 直接对应于您指定的年终YuanCloud日期。因此,任何时候你生成 损益表 ,开始日期与 * 财政年度* [UNKNOWN NODE problematic]一致, 科目余额都是0。

    一旦会计创建凭证用于分配* 当年利润 ,你应该设置本财政年度的最后一天为 锁定日期 。确保在这样做之前,您确认是否当前年度收入 [UNKNOWN NODE problematic]资产负债表 * [UNKNOWN NODE problematic]中正确余额为0。
